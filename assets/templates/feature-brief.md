# Feature Brief 模板

> 用途：一句话或小型需求的迷你 PRD。
> 落地位置：复制本文件到 `docs/features/<feature-kebab-name>/feature-brief.md`，
> 或在 PR 描述中直接填写本结构。

## 1. 背景与目标

- **背景**：为什么要做这件事？现状 / 用户痛点 / 业务诉求是什么？
- **目标**：完成后期望达到的状态。可度量优先（例如“减少 X 接口 50% 错误率”）。

## 2. 范围

- 包含什么：
- 不包含什么（明确反向边界）：

## 3. 用户场景

- 谁在什么场景下使用？典型操作路径是什么？

## 4. 接口 / 数据 / 配置影响

| 维度 | 是否影响 | 说明 |
| --- | --- | --- |
| 新增 / 修改 API | 是 / 否 | 列出接口路径与变更点 |
| 数据库变更 | 是 / 否 | 涉及的表、字段、索引；是否需要 Flyway 迁移 |
| 鉴权 / 权限 | 是 / 否 | 是否新增菜单 / 角色 / 数据权限 |
| 配置 | 是 / 否 | 新增 Nacos 配置项 / 环境变量 |
| 缓存 / 搜索 | 是 / 否 | 是否新增缓存 key、ES 索引或字段 |
| 第三方 / 外部依赖 | 是 / 否 | 是否新增 SDK 或调用 |

## 5. 影响风险（IMPACT_ANALYSIS）

逐项检查，避免漏看：

- [ ] 是否影响其他业务流程？
- [ ] 是否改变已有接口语义或返回结构？
- [ ] 是否影响导出 / 统计 / 报表？
- [ ] 是否影响搜索 / 分页 / 过滤？
- [ ] 是否影响缓存 / 索引 / 锁？
- [ ] 是否影响租户隔离？
- [ ] 是否影响测试覆盖？
- [ ] 是否影响文档（Swagger / 模块文档）？

> 出现以下任一情况时，**升级到 `dev-flow-doc-*` 完整流程**：
> - 涉及 DB 新增 / 改表 / 改索引
> - 涉及鉴权 / 数据权限 / 租户隔离
> - 改变已有接口语义或返回结构
> - 影响多个业务流程或多个入口
> - 涉及金额 / 订单 / 支付 / 审核 / 钱包等高风险业务
> - 当前测试覆盖不足且改动风险较高
> - 预估超过 8 工时

```
接口路径：GET /api/admin/xxx
鉴权：    后台用户登录
请求参数：
  - foo: Long  必填  说明
  - bar: String 可选  说明
返回：     R<XxxVO>
错误：
  - 400 参数错误
  - 401 未登录
  - 403 无权限
  - 404 资源不存在
兼容性：   不影响已有调用方 / 已通知调用方
```
