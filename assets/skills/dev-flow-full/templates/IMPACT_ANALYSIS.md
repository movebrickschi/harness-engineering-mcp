# 影响分析文档

> 目的：识别本次改动可能影响的现有模块，提前规划回归测试范围。

## 1. 改动概览

| 改动类型 | 数量 | 风险等级 |
|---------|------|---------|
| 新增文件 | N | 低 |
| 修改文件 | M | 中 |
| 删除文件 | 0 | - |
| 数据库迁移 | 1 | 中 |
| API 新增 | X | 低 |
| API 修改 | 0 | - |
| API 删除 | 0 | - |

## 2. 反向依赖链

### 改动文件：src/admin/admin.module.ts
**被谁依赖**：
- src/app.module.ts → 导入 AdminModule
- src/admin/*/* → 同模块下的子模块

**改动内容**：仅新增 SuppliersModule 导入
**风险**：极低（纯新增，无修改既有逻辑）
**回归范围**：所有 admin 后台页面冒烟测试

### 改动文件：prisma/schema.prisma
**被谁依赖**：
- 所有使用 Prisma Client 的 Service
- 自动生成的 @prisma/client 类型

**改动内容**：新增 Supplier、ApprovalLog model
**风险**：中（schema 变更影响 Prisma Client 类型生成）
**回归范围**：
- 全量 prisma generate 后跑类型检查
- 所有 Service 单测
- 数据库迁移在 staging 环境先验证

- 数据保留：审核日志保留 30 天后清理
- 预计回滚耗时：10 分钟
